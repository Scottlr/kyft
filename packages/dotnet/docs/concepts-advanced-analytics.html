Concepts

Ask deeper interval questions.

Once windows are aligned, Spanfold can measure agreement, divergence, timing, containment, coverage, point-in-time matches, and live finality without hand-written interval joins.

Problem

Advanced questions usually start as interval joins.

A provider comparison, SLA audit, or incident review rarely needs one number. It needs to know which source was active, which source was silent, which ranges overlapped, where coverage disappeared, and what was knowable when a decision was made.

Hand-written interval joins tend to mix those concerns together: filtering, clipping, source alignment, null handling, row labeling, finality, and summary math all end up in one query. Spanfold keeps the interpretation step separate. First record windows with stable ids and context. Then compose comparator row families over those windows.

The result is still granular. Coverage ratios and lead/lag summaries are derived from rows that carry the original window ids, ranges, source names, segments, tags, and finality.

Journey

From source windows to decision rows.

The advanced analytics flow is deliberately staged: select the two sides, normalize them onto a temporal axis, align the ranges, emit the row families needed by the decision, then aggregate only after the evidence rows exist.

1. Align lanes

Target primary active
Against backup active

Select source, key, window, segment, and tag scope before asking any analytic question.

2. Emit evidence

Rows residual overlap missing
Timing target leads

Comparator rows name the kind of evidence rather than hiding it inside a single aggregate.

3. Clip live state

Open still active
Result provisional

Live runs evaluate open windows at a declared horizon and mark rows whose answer can still change.

Solution

Compose the evidence rows you need.

Start with the row families that answer the business question. Add overlap when you need agreement, residual when the target must be covered, missing when the comparison source reported something the target did not, and coverage or gap rows when the output must roll up into a threshold. 

var qa = pipeline.History
    .Compare("Provider QA")
    .Target("provider-a", s => s.Source("provider-a"))
    .Against("provider-b", s => s.Source("provider-b"))
    .Within(scope => scope
        .Window("DeviceOffline")
        .Tag("region", "eu-west"))
    .Normalize(n => n.RequireClosedWindows().OnPosition())
    .Using(c => c
        .Overlap()
        .Residual()
        .Missing()
        .Coverage()
        .Gap()
        .Containment()
        .LeadLag(
            LeadLagTransition.Start,
            TemporalAxis.ProcessingPosition,
            toleranceMagnitude: 5))
    .Run();

Analytics

Choose the row family that matches the decision.

Coverage

Use when the target should have been backed by another source. Coverage rows support ratios such as covered magnitude divided by target magnitude, while preserving the ranges that contributed to the number.

Gaps

Use when absence is the failure mode. Gap rows isolate uncovered spans so alerts and audits can point to concrete start and end positions rather than just a failed percentage.

Containment

Use when one side is expected to fully wrap another, such as a maintenance window containing a provider outage or a parent state containing child activity.

Lead and lag

Use when transition timing matters. Compare starts or ends on processing position or event time, apply a tolerance, and classify whether the target led, lagged, or matched.

As-of

Use for audits and replay. Evaluate the previous or next matching window at a decision point after applying known-at normalization, so future evidence does not leak into the answer.

Live finality

Use when open windows are part of the answer. Clip them to a horizon and keep provisional rows visibly separate from final rows in exports and summaries.

Examples

Turn evidence rows into operational answers.

Coverage threshold

var coverage = pipeline.History
    .Compare("Backup coverage SLA")
    .Target("primary", s => s.Source("primary"))
    .Against("backup", s => s.Source("backup"))
    .Within(scope => scope.Window("DeviceOffline"))
    .Using(c => c.Coverage().Gap())
    .Run();

var covered = coverage.CoverageRows.TotalCoveredMagnitude();
var expected = coverage.CoverageRows.TotalTargetMagnitude();
var ratio = expected == 0 ? 1d : (double)covered / expected;

var longestGap = coverage.GapRows
    .OrderByDescending(row => row.Range.GetPositionLength())
    .FirstOrDefault();

Transition drift

var timing = pipeline.History
    .Compare("Provider start drift")
    .Target("provider-a", s => s.Source("provider-a"))
    .Against("provider-b", s => s.Source("provider-b"))
    .Within(scope => scope.Window("DeviceOffline"))
    .Using(c => c.LeadLag(
        LeadLagTransition.Start,
        TemporalAxis.ProcessingPosition,
        toleranceMagnitude: 5))
    .Run();

var lateStarts = timing.LeadLagRows
    .Where(row => row.Direction == LeadLagDirection.TargetLags)
    .ToArray();

Point-in-time audit

var audit = pipeline.History
    .Compare("Decision audit")
    .Target("decision", s => s.Source("decision-service"))
    .Against("risk", s => s.Source("risk-provider"))
    .Within(scope => scope.Window("HighRisk"))
    .Normalize(n => n.KnownAtPosition(42))
    .Using(c => c.AsOf(
        AsOfDirection.Previous,
        TemporalAxis.ProcessingPosition,
        toleranceMagnitude: 10))
    .Run();

var decisionEvidence = audit.AsOfRows
    .Where(row => row.TargetPoint.Position == 42);

Live dashboard export

var horizon = TemporalPoint.ForPosition(1_250);

var live = pipeline.History
    .Compare("Current provider QA")
    .Target("provider-a", s => s.Source("provider-a"))
    .Against("provider-b", s => s.Source("provider-b"))
    .Within(scope => scope.Window("DeviceOffline"))
    .Normalize(n => n.ClipOpenWindowsTo(horizon))
    .Using(c => c.Overlap().Residual().Missing())
    .RunLive(horizon);

var provisionalRows = live.ProvisionalRowFinalities();

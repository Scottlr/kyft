Concepts

How Spanfold turns state windows into analysis.

Start with recorded windows, then choose the lens: roll child state up to parent state, collapse many sources into a cohort, compare every provider pair, or run advanced interval analytics.

Map

Pick the concept that matches the question.

Roll-ups Aggregate child windows into parent windows while preserving the segment context that still matters. Segments and tags Split windows on analytical boundaries while keeping descriptive metadata attached. Cohorts Compare one source against a group treated as a single derived lane. Source matrix Read directional pairwise comparisons across a provider set. Hierarchy Explain parent roll-up activity from child contribution windows. Live finality Clip open windows at a horizon and separate provisional rows from final evidence. Advanced analytics Use coverage, gap, containment, lead/lag, as-of, finality, and row aggregations for deeper questions.

Mental model

Record first, analyze second.

Spanfold does not ask every comparator to re-derive state. The runtime records the time ranges where predicates were active. Analysis then works over those ranges: normalization, alignment, comparison, export, and explanation.

That split keeps the concepts composable. A roll-up can produce parent windows that are compared later. A cohort can become the comparison side. A source matrix can run the same plan across every provider pair.

C# temporal analysis package

Record state as windows. Compare what each source saw.

Kyft is for systems where the important question is not just the latest value. It records when state was active, which lane observed it, where sources overlapped or diverged, and what was knowable at a point in time.

GitHub API reference

The problem

Window analytics becomes hard when state has multiple histories.

A current-state table collapses the answer. Kyft keeps the temporal evidence visible: the active ranges, the source lanes, the gaps, and the horizon used to make the result.

Package shape

Small primitives for temporal evidence.

Define a keyed predicate, record the resulting windows, then query or compare those windows. Kyft stays focused on temporal state history rather than becoming a stream processor, metrics platform, or database abstraction.

Minimal example

Define a state, ingest events, compare lanes.

using Kyft;

var pipeline = Kyft
    .For<DeviceSignal>()
    .RecordWindows()
    .TrackWindow(
        "DeviceOffline",
        key: signal => signal.DeviceId,
        isActive: signal => !signal.IsOnline);

pipeline.Ingest(new DeviceSignal("device-17", false), source: "source-a");
pipeline.Ingest(new DeviceSignal("device-17", true), source: "source-a");

var result = pipeline.History
    .Compare("Source comparison")
    .Target("source-a", selector => selector.Source("source-a"))
    .Against("source-b", selector => selector.Source("source-b"))
    .Within(scope => scope.Window("DeviceOffline"))
    .Using(comparators => comparators.Overlap().Residual().Missing())
    .Run();

public sealed record DeviceSignal(string DeviceId, bool IsOnline);

C# temporal analysis package

State changes are easy. State over time is the hard part.

Kyft records temporal state windows and compares them across sources, providers, or pipeline stages, so you can see who observed a state, who missed it, where histories overlapped, and what was knowable at a specific point in time.

What it does

Kyft turns event predicates into evidence you can inspect.

In many systems, the latest value is not enough. You need to know when a state was active, which lane reported it, whether another source agreed, and how that answer changes at a historical horizon.

A temporal window is a recorded period of state.

A window opens when a predicate becomes true for a key. It closes when the predicate becomes false, a boundary splits the analysis, or a live horizon clips the current view.

Why it exists

Ordinary state tracking throws away the timeline.

A current-state table can tell you that a device is healthy now. It cannot tell you that one monitoring provider reported it offline for ten minutes while another saw only five, or that the comparison source went silent during the period you cared about.

Kyft keeps the temporal record as first-class data. It lets C# systems compare observed histories without becoming a general stream processor, metrics backend, or database abstraction.

Concrete use cases

Use Kyft when the question depends on time, source, and evidence.

Monitoring provider comparison

Find where provider A reported an outage that provider B missed, overlapped, or reported late.

Decision-point audit

Evaluate only the windows known at a specific position or timestamp, without future leakage.

Live open-window analysis

Run comparisons against still-open windows and separate provisional rows from final rows.

Pipeline history analysis

Compare stages in a data pipeline to see where state diverged, lagged, or left gaps.

Key capabilities

Small primitives, serious temporal questions.

Record windows

Capture when keyed state opens and closes, including source, partition, segment, and tag context.

Compare histories

Emit overlap, residual, missing, coverage, gap, containment, lead/lag, and as-of rows.

Reason at a horizon

Ask what was open or knowable at a processing position or timestamp.

Handle live analysis

Clip ongoing windows to an explicit horizon and mark dependent comparison rows as provisional.

Explain and export

Produce deterministic JSON, JSON Lines, Markdown, and self-contained debug HTML artifacts.

Test recorded history

Use optional testing helpers for compact fixtures and assertions without running a full pipeline.

Install and code

Define the state once, then query and compare the recorded history.

Install

dotnet add package Kyft
dotnet add package Kyft.Testing

`Kyft.Testing` is optional. It is useful for compact comparison fixtures in consumer test suites.

Track windows

using Kyft;

var pipeline = Kyft
    .For<DeviceSignal>()
    .RecordWindows()
    .TrackWindow(
        "DeviceOffline",
        key: signal => signal.DeviceId,
        isActive: signal => !signal.IsOnline);

pipeline.Ingest(new DeviceSignal("device-17", false), source: "provider-a");
pipeline.Ingest(new DeviceSignal("device-17", true), source: "provider-a");

public sealed record DeviceSignal(string DeviceId, bool IsOnline);

Compare two sources

var result = pipeline.History
    .Compare("Provider QA")
    .Target("provider-a", selector => selector.Source("provider-a"))
    .Against("provider-b", selector => selector.Source("provider-b"))
    .Within(scope => scope.Window("DeviceOffline"))
    .Using(comparators => comparators
        .Overlap()
        .Residual()
        .Missing()
        .Coverage())
    .Run();

foreach (var row in result.ResidualRows)
{
    Console.WriteLine($"{row.WindowName} {row.Key} target-only {row.Range.Start}..{row.Range.End}");
}

Why not just X?

Kyft is focused on recorded temporal state, not replacing your platform.

Storing timestamps in a database

A database can persist windows, but it will not give you staged comparison plans, source selectors, normalization, live finality, diagnostics, or deterministic exports by itself.

Latest-state tracking

Latest-state tables answer what is true now. Kyft answers when it was true, who saw it, and whether another source missed it.

Generic event processing

Stream processors route and aggregate events. Kyft stays smaller: it records state windows and compares their temporal evidence inside C# code.

Metrics dashboards

Dashboards compress time into counters, rates, and charts. Kyft keeps the individual windows and emits rows you can audit.

Event sourcing

Event sourcing preserves facts and rebuilds state. Kyft analyzes the windows produced after those facts have been interpreted.

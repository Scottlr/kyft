This page documents the main package surface for building pipelines, recording window history, comparing sources, exporting artifacts, and reasoning at live or historical horizons. The NuGet package also ships XML documentation generated from the public comments. 

dotnet add package Kyft
dotnet add package Kyft.Testing

Pipeline Creation

Start with `Kyft.For<TEvent>()`, define one or more windows, then build an `EventPipeline<TEvent>`.

Kyft.For<TEvent>()Creates an event pipeline builder for the event type.
EventPipelineBuilder<TEvent>.RecordWindows()Enables recording of open and closed windows during ingestion.
WithEventTime(...)Configures event timestamps for timestamp-based comparisons and exports.
TrackWindow(...)Builds a pipeline for one state-driven source window.
Build()Builds an event pipeline from configured windows and options.

Window Definitions

A window opens when its active predicate becomes true for a key and closes when it becomes false or a boundary splits the recorded history.

Window(...)Adds an independent source window to a pipeline builder.
WindowDefinitionBuilder.Key(...)Selects the logical key that owns independent state.
ActiveWhen(...)Defines the predicate that keeps the window active.
Segment(...)Adds a boundary-aware analytical dimension that can split windows when the value changes.
Tag(...)Captures descriptive metadata when a window opens.
OnOpened(...) / OnClosed(...)Registers transition callbacks.

Recorded History

`EventPipeline.History` is the main surface for direct history queries, annotations, snapshots, and comparison builders.

WindowHistory.ClosedWindowsMaterializes closed windows recorded by the pipeline.
WindowHistory.OpenWindowsMaterializes windows still open in runtime state.
Query()Starts a read-only direct query over recorded windows.
SnapshotAt(TemporalPoint)Evaluates recorded history at a horizon without mutating runtime state.
Annotate(...)Attaches append-only metadata to open or closed windows.
AnnotationsKnownAt(...)Reads annotations that were knowable at or before a horizon.
var open = pipeline.History.Query()
    .Window("DeviceOffline")
    .Lane("provider-a")
    .OpenWindowsAt(TemporalPoint.ForPosition(100));

Comparison Builder

The comparison builder is staged so the question remains readable: target, comparison side, scope, normalization, comparator, execution.

Compare(name)Starts a staged comparison over recorded history.
Target(name, selector => ...)Sets the baseline side.
Against(name, selector => ...)Adds one comparison side.
AgainstCohort(name, cohort => ...)Collapses a group of sources into one derived comparison side.
Within(scope => ...)Limits the comparison to a window, segment, tag, or full scope.
Using(comparators => ...)Selects the comparator row families to emit.
Run() / RunLive(...)Executes historical or live comparison.

Normalization And Selection

Normalization makes temporal rules explicit before alignment and comparator execution.

RequireClosedWindows()Rejects open windows for historical comparison.
ClipOpenWindowsTo(horizon)Clips open windows to a live evaluation horizon.
OnPosition() / OnEventTime()Selects the temporal axis used for alignment.
KnownAtPosition(...) / KnownAt(...)Filters to windows knowable at a decision point.
CoalesceAdjacentWindows()Coalesces adjacent windows when analysis requires that policy.
RejectDuplicateWindows()Fails validation when duplicate windows are present.

Comparators And Result Rows

Comparator rows are structured output for auditing overlap, divergence, coverage, and timing drift.

Overlap()Rows where target and comparison windows are active at the same time.
Residual()Target-only rows: what the comparison missed.
Missing()Comparison-only rows: what the target did not see.
Coverage()Coverage rows and summaries by target magnitude.
Gap()Empty spaces inside an observed scope envelope.
SymmetricDifference()Target-only and comparison-only disagreement in one row family.
Containment()Whether target windows stay inside comparison windows.
LeadLag(...) / AsOf(...)Transition drift and point-in-time lookup.

Exports And Debugging

Exports are intended for workflow boundaries, tests, issue reports, notebooks, and support handoff rather than ingestion hot paths.

ExportJson()Writes deterministic JSON for plans or results.
ExportJsonLines()Streams result rows as JSON Lines.
ExportMarkdown()Writes deterministic Markdown explanations.
Explain()Renders deterministic diagnostic text.
ExportDebugHtml()Creates a self-contained visual timeline and segment graph.
ComparisonDebugHtmlOptions.ToFile(...)Lets configuration decide whether `Run()` or `RunLive()` writes a debug artifact.

Temporal Types

Kyft supports processing-position analysis and timestamp analysis with explicit temporal types.

TemporalPoint.ForPosition(long)Creates a processing-position horizon or comparison point.
TemporalPoint.ForTimestamp(DateTimeOffset, clock)Creates a timestamp-based temporal point.
TemporalRange.Closed(start, end)Creates a half-open closed range with a known end.
TemporalRange.Open(start)Creates an open range for live or ongoing state.
GetPositionLength() / GetTimeDuration()Measures ranges on the appropriate axis.

Lane Liveness

`LaneLivenessTracker` converts sparse observations into liveness signals that can be recorded as ordinary Kyft windows.

LaneLivenessTracker.ForLanes(...)Creates a deterministic liveness tracker for expected lanes.
Observe(lane, observedAt)Records that a lane reported at a timestamp.
Check(horizon)Emits no-signal or recovered signals at a horizon.
LaneLivenessSignalEvent type suitable for recording silence windows with a normal Kyft pipeline.

Testing Helpers

`Kyft.Testing` is a separate optional package for consumer test suites.

WindowHistoryFixtureBuilderBuilds compact recorded histories without running a full event pipeline.
KyftAssertFramework-neutral assertions for comparison artifacts.
KyftSnapshotSmall snapshot helpers for Kyft JSON and Markdown artifacts.
